# Changelog v2.2.3 -- 2026-05-31

## CTAN Resubmission (Reviewer Feedback)

### Changed

- **Doctype configs**: Renamed 27 files from `{name}.sty` to `omnilatex-{name}.sty` to avoid filename conflicts with other CTAN packages.
- **Lua runtime**: Renamed `luaotfload-workaround.lua` to `omnilatex-luaotfload-workaround.lua`.
- **CTAN zip**: Removed TDS zip generation (per CTAN reviewer: "package is complicated enough without it").
- **Documentation PDF**: Renamed from `omnilatex.pdf` to `omnilatex-doc.pdf`, placed in `doc/` inside package directory.
- **Documentation source**: Now uses `doc/omnilatex.tex` (self-contained 315-line manual) instead of `main.tex` (thesis template referencing missing content files).
- **Bibliography**: Moved from `bib/` to `doc/bib/` in CTAN zip (sample bibliography is documentation, not runtime).
- **README**: Clarified CTAN listing vs. TeX Live inclusion distinction. Added manual install instructions.

### Fixed

- **Tests**: Resolved CTAN test vs. shell script divergence (Phase 1.5). `test_ctan.py` now asserts institutions ARE present (matching `make-ctan-zip.sh`), excludes test-univ/generic, and uses correct `ctan/` output path.

## Color Theme System

### Fixed

- **Headings**: Section/chapter/caption colors now respond to mid-document `\usetheme{}` calls. Previously only page background changed. Fix uses `\addtokomafont` with dynamically-resolved `\color{}` instead of preamble-only `\setkomacolor`.
- **Block environments**: Redefined as tcolorbox environments unconditionally (not just on TL2025+). Blocks now correctly pick up theme colors mid-document.
- **tcbuselibrary leak**: Moved `\tcbuselibrary{skins}` from `\AddToHook{begindocument}` to preamble. Previously caused `] 1 ] 1 skins` artifact tokens in PDF output.

## Beamer

### Changed

- **Templates**: Rewrote `beamer-academic` (conference talk with tikz diagrams, code listings, equations), `beamer-corporate` (business review with KPI dashboard, deployment pipeline, risk register), and `beamer-defense` (thesis defense with Lean 4 architecture, proof statistics, reproducibility theorem).
- **Themes added**: Berlin, Copenhagen, Singapore, Goettingen, Rochester, Pittsburgh, Berkeley, PaloAlto (previously only Madrid, Warsaw, metropolis).
- **Command clash**: Renamed OmniLaTeX's theme applier from `\usetheme` to `\omnilatexusetheme`. `\usetheme` now uses `\providecommand` to avoid clashing with beamer's built-in theme command.

### Fixed

- **Beamer/beamer-module filenames**: Updated to new `omnilatex-` prefix for consistency.

## Institutions

### Fixed

- **Pattern B integration**: aalto, chalmers, kit, uoft, ntnu now define `universityprimary`/`universitysecondary` color aliases. Previously used institution-specific names that were invisible to the theme system.
- **Dead logo paths**: 10 institutions referenced non-existent `assets/images/institutions/` directory. Fixed to `assets/logos/{name}/logo` convention.
- **Missing sections**: Added logo templates to columbia and harvard (previously the thinnest configs).

### Removed

- **Orphaned macros**: Removed unused `\instLogo{}` and `\instURL{}` from Pattern B configs (never consumed by any core code).
- **Vestigial color blocks**: Removed commented-out `\colorlet{colorTitleBack/Section/Link/TableRule/Footer}` from all 20+ institution configs (202 lines deleted).

## CI/CD

### Changed

- **Test standardization**: All 5 CI platforms now run the same 8 test files with `--timeout=60`.
- **GitHub Actions**: Added `lint` job (black, isort, flake8) to `build.yml`.
- **Forgejo**: Pinned all workflow actions by SHA (security hardening).
- **Concurrency**: Fixed static concurrency groups in `integration-matrix.yml`, `build-examples.yml`, `docs.yml` to include `${{ github.ref }}`.
- **Path triggers**: Added `lua/**` to `integration-matrix.yml`.
- **Cross-platform**: Removed no-op Windows job that consumed runner minutes.
