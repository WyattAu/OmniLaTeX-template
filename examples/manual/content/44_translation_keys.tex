\chapter{Translation Key Reference}
\label{ch:translation-keys}

OmniLaTeX uses the \pkg{translator} package to manage localised
strings for all 18 supported languages.  Every caption, heading, and
label is controlled by a \emph{translation key}.  This chapter
documents all 47 keys, their English values, and their purpose.

Keys are resolved through a fallback chain: language-specific
definition~$\to$~English, terminating at the fixed-point
\texttt{fallback(english) = english}.  This ensures every key
resolves to a defined string regardless of language support.

\section{Adding New Keys}
\label{sec:tl-add}

To add a new translation key, register it in
\texttt{lib/language/omnilatex-i18n.sty}:

\begin{verbatim}
\DeclareTranslation{english}{MyNewKey}{Default English Value}
\DeclareTranslation{german}{MyNewKey}{Deutsche Übersetzung}
\DeclareTranslation{french}{MyNewKey}{Traduction française}
% ... repeat for all 18 languages
\end{verbatim}

Then use \cs{GetTranslation\{MyNewKey\}} anywhere in the document to
retrieve the current language's value.  If a language does not define
\texttt{MyNewKey}, the fallback chain resolves to the English
definition automatically.

Keys follow the PascalCase convention and should use descriptive
names that indicate their context (e.g., \texttt{Contfoot} for
``continued on next page'' rather than \texttt{CTF}).

\end{Verbatim}
