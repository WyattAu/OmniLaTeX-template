\chapter{Git Integration}
\label{ch:git}

\section{Git Metadata Commands}

The template provides four macros for embedding repository metadata in the
document, all defined in \texttt{omnilatex-utils.sty}:

\begin{description}
  \item[{\textbackslash GitRefName}] The current branch or tag name.
        \begin{minted}{latex}
        Compiled from branch: \GitRefName
        \end{minted}

  \item[{\textbackslash GitShortSHA}] The abbreviated commit hash (8 characters).
        \begin{minted}{latex}
        Commit: \GitShortSHA
        \end{minted}

  \item[{\textbackslash GitLongSHA}] The full 40-character commit hash.
        \begin{minted}{latex}
        Full SHA: \GitLongSHA
        \end{minted}

  \item[{\textbackslash BuildDate}] The date and time of compilation.
        \begin{minted}{latex}
        Built on \BuildDate.
        \end{minted}
\end{description}

All Git macros require \texttt{shell-escape} to be enabled, which is the
default in all build modes.  The macros invoke \texttt{git} at compile
time, so the working directory must be inside a Git repository.

\section{Verification Box}

\cs{GitVerificationBox} renders a formatted box suitable for colophon pages:

\begin{minted}{latex}
\chapter*{Colophon}
\GitVerificationBox
\end{minted}

The box displays:
\begin{itemize}
  \item The full and short commit SHA.
  \item The branch or tag name.
  \item A verification command the reader can run to confirm the build.
\end{itemize}

\section{Build Date}

\cs{BuildDate} integrates with \texttt{datetime2} (\cs{DTMnow}) to produce
a locale-aware timestamp:

\begin{minted}{latex}
This document was compiled on \BuildDate.
\end{minted}

The format respects the document's language and date settings configured
via the \texttt{mainlang} class option.

\section{.gitignore for LaTeX Projects}

A well-configured \texttt{.gitignore} keeps the repository clean by
excluding generated files.  OmniLaTeX repositories should include the
following patterns:

\begin{minted}{gitignore}
# LaTeX auxiliary files
*.aux
*.lof
*.lot
*.loe
*.lol
*.lor
*.toc
*.out
*.synctex.gz
*.fdb_latexmk
*.fls
*.idx
*.ilg
*.ind
*.ist
*.acn
*.acr
*.alg
*.glg
*.glo
*.gls
*.glsdefs
*.slg
*.slo
*.sls

# Bibliography and glossary intermediates
*.bbl
*.bcf
*.blg
*.run.xml
*-*.glstex
*.glstex

# Build output
*.pdf
build/

# Minted cache
_minted-*/
__pycache__/
\end{minted}

The pattern \texttt{*-*.glstex} is required because
\texttt{glossaries-extra} generates numbered variants
(\texttt{main-1.glstex}, \texttt{main-2.glstex}, \ldots) when multiple
glossary types are present.

\section{Binary PDF Handling}

PDFs generated by the build should generally be excluded from version
control.  However, some workflows require committing specific release
PDFs.  Use \texttt{.gitattributes} to control diff behaviour:

\begin{minted}{gitattributes}
# Binary diff (default)
*.pdf binary

# Force text diff on .tex files regardless of line endings
*.tex text diff=latex
\end{minted}

Git's built-in binary diff marks PDFs as changed without showing content
differences.  To compare PDFs visually, use an external diff driver such
as \texttt{diff-pdf}:

\begin{minted}{gitconfig}
[diff "pdf"]
  textconv = diff-pdf
  binary = true
\end{minted}

\section{.latexmkrc in Version Control}

The root \texttt{.latexmkrc} should be committed to the repository so
that all collaborators and CI pipelines share identical build settings:

\begin{minted}{bash}
# Verify .latexmkrc is tracked
git ls-files .latexmkrc
\end{minted}

When \texttt{build.py} creates example builds, it symlinks the root
\texttt{.latexmkrc} into each example directory.  This ensures that
standalone \texttt{latexmk} invocations use the same configuration:

\begin{minted}{bash}
# build.py creates this symlink automatically
ln -s ../../.latexmkrc examples/my-example/.latexmkrc
\end{minted}

\section{Pre-Commit Hooks}

Pre-commit hooks catch errors before they reach the repository.  Two
common hooks for LaTeX projects are \texttt{chktex} (syntax checking)
and \texttt{latexmk -c} (auxiliary file cleanup):

\begin{minted}{bash}
#!/usr/bin/env bash
# .git/hooks/pre-commit

echo "Running chktex..."
find . -name '*.tex' -not -path './build/*' \
  -exec chktex -q -n 1 -n 44 {} + || true

echo "Cleaning auxiliary files..."
latexmk -c -silent 2>/dev/null || true
\end{minted}

For a more robust setup, use the \texttt{pre-commit} framework with
community-maintained hooks:

\begin{minted}{yaml}
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v5.0.0
    hooks:
      - id: trailing-whitespace
        files: \.tex$
      - id: end-of-file-fixer
        files: \.tex$
  - repo: https://github.com/cmhughes/latexindent.pl
    rev: v3.24.4
    hooks:
      - id: latexindent
\end{minted}

\section{Collaborative Editing}

When multiple authors work on the same document, follow these practices:

\begin{enumerate}
  \item Divide content into separate \texttt{.tex} files and assemble
        them via \verb|\input{}| or \verb|\import{}| in the main file.
  \item Use \texttt{git merge} rather than rebasing to preserve a
        linear history for review.
  \item Resolve merge conflicts promptly; \LaTeX{} syntax errors from
        malformed merges are difficult to diagnose.
  \item Commit the \texttt{flake.lock} file when using Nix to ensure
        all collaborators share identical tool versions.
\end{enumerate}

\section{Release Tagging}

Tag release versions in Git and let the build system embed the tag name
via \cs{GitRefName}:

\begin{minted}{bash}
git tag -a v1.0.0 -m "Thesis final submission"
git push origin v1.0.0
BUILD_MODE=prod python build.py
\end{minted}

The resulting PDF will display \texttt{v1.0.0} as the branch/tag name
on the colophon page.

\section{.gitattributes for Line Endings}

LaTeX source files should use LF line endings consistently.  Configure
\texttt{.gitattributes} to enforce this:

\begin{minted}{gitattributes}
# Force LF for all text files
* text=auto eol=lf

# Binary formats
*.pdf binary
*.png binary
*.jpg binary
*.svg binary
\end{minted}

This prevents CRLF/LF mismatches that cause spurious diffs on Windows
systems.
