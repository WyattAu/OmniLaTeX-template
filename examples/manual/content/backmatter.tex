\section*{Compilation Sequence}

The manual is compiled using the standard OmniLaTeX build sequence.
The recommended build command is:

\begin{minted}{latex}
% Full build (four passes for cross-references)
lualatex manual.tex
biber manual
lualatex manual.tex
lualatex manual.tex
\end{minted}

The build sequence resolves cross-references, bibliography entries,
glossary entries, and index entries across multiple passes:

\begin{enumerate}
    \item \textbf{First pass} -- collects citation keys, glossary references,
          and index entries into auxiliary files.
    \item \textbf{Biber} -- processes \texttt{.bcf} to generate the
          bibliography database (\texttt{.bbl}).
    \item \textbf{Second pass} -- incorporates bibliography and resolves
          forward references.
    \item \textbf{Third pass} -- ensures all cross-references (equations,
          figures, tables, glossary, index) are fully resolved.
\end{enumerate}

For iterative editing, two passes (lualatex + lualatex) are usually
sufficient.  A full four-pass build is recommended before final submission
or publication.

\section*{Reproducible Builds with Nix}

OmniLaTeX supports fully reproducible builds via Nix.  When using the Nix
flake, all system dependencies (TeX~Live, Python, Pygments) are pinned to
exact versions, ensuring that the output is bit-for-bit identical across
machines.  See Chapter~\ref{ch:nix-workflow} for detailed Nix workflow instructions.

\begin{minted}{latex}
% Reproducible build via Nix
nix build
# Output: result/bin/manual.pdf
\end{minted}

The Nix flake pins all dependencies using \texttt{flake.lock}.  To update
dependencies while maintaining reproducibility:

\begin{minted}{latex}
nix flake update        # Update all inputs
nix flake lock --update-input nixpkgs  # Update specific input
\end{minted}

\section*{Bibliography Details}

The bibliography in this manual is managed by \ctanpackage{biblatex} with the
Biber backend.  Two bibliography sections are printed:

\begin{enumerate}
    \item \textbf{Cited Works} -- entries referenced via
          \verb|\cite{}| in the text.  These appear with numbered labels.
    \item \textbf{Further Reading} -- all other entries in the
          \texttt{.bib} file that were not explicitly cited.  These appear
          without numbering (\texttt{env=bibnonum}) and are intended as
          supplementary references.
\end{enumerate}

The \verb|\nocite{*}| command ensures that all entries from the
bibliography database are included, regardless of whether they were cited
in the text.  Remove this command to list only cited works.

\section*{Glossary and Index}

The glossary is generated by \ctanpackage{glossaries-extra} and collects
three categories of entries:

\begin{description}
    \item[Symbols] Mathematical symbols with their notation, description,
        and units (e.g.\ velocity, pressure, temperature).
    \item[Abbreviations] Acronyms and initialisms that are expanded on
        first use (e.g.\ CJK, RTL, DOI).
    \item[General terms] Technical terms that benefit from a concise
        definition.
\end{description}

The index is generated by \ctanpackage{imakeidx} and uses the
\texttt{bookindex} style for two-level entries.  Index entries are
embedded in the source using \verb|\index{term}| and
\verb|\index{term!subterm}| commands.  The \texttt{printunsrtindex}
command renders the index in unsorted order (as written in the source).
