\chapter{Build Modes}
\label{ch:buildmodes}

OmniLaTeX supports three build modes that trade completeness for speed.
The active mode is determined by the \texttt{BUILD\_MODE} environment
variable and is communicated to the \LaTeX{} source via macros defined
in \texttt{.latexmkrc}.

\section{dev Mode}

Development mode (\texttt{BUILD\_MODE=dev}) is the default.  It runs
\texttt{latexmk} with up to six passes, bibliography processing, and
\texttt{shell-escape} enabled:

\begin{minted}{bash}
BUILD_MODE=dev python build.py
\end{minted}

This mode is optimised for iteration speed while still producing a correct
PDF with resolved cross-references and citations.  Biber runs without
\texttt{-{}-validate-datamodel} to avoid slow validation on every
iteration.  Glossary compilation via \texttt{bib2gls} is active.

\section{prod Mode}

Production mode (\texttt{BUILD\_MODE=prod}) performs full validation:

\begin{minted}{bash}
BUILD_MODE=prod python build.py
\end{minted}

All packages are active, including accessibility checks (\cs{validatstructure}),
glossary compilation, and minted code highlighting.  Biber runs with
\texttt{-{}-validate-datamodel} to catch malformed bibliography entries.
This is the mode to use for final output and archival builds.

\section{ultra Mode}

Ultra mode (\texttt{BUILD\_MODE=ultra}) prioritises speed over completeness:

\begin{minted}{bash}
BUILD_MODE=ultra python build.py
\end{minted}

Only a single \texttt{latexmk} pass is executed.  Bibliography processing
is disabled entirely (\texttt{\$bibtex\_use = 0}) and Biber is replaced
with \texttt{true}.  Glossary compilation is skipped.  Cross-references
will show ``??'' placeholders.  Use this mode for rapid content drafting
where exact output is not yet needed.

\section{Mode Detection}

Document source can detect the current build mode via three macros:

\begin{minted}{latex}
\ifdefined\OmniDev
  % Development build -- include draft notes
  \todo{Check this section before submission.}
\fi

\ifdefined\OmniProd
  % Production build -- run all validation
  \validatstructure
\fi

\ifdefined\OmniUltra
  % Ultra-fast build -- skip heavy packages
\fi
\end{minted}

The \texttt{BUILD\_MODE} environment variable is also available in shell
commands invoked via \texttt{shell-escape}.

\section{Setting Build Mode}

Two methods are available:

\begin{enumerate}
  \item \textbf{Environment variable:}
    \begin{minted}{bash}
    BUILD_MODE=prod python build.py
    \end{minted}

  \item \textbf{Command-line flag:}
    \begin{minted}{bash}
    python build.py --mode prod
    \end{minted}
\end{enumerate}

The command-line flag takes precedence over the environment variable.
If neither is set, \texttt{dev} is used.

\section{.latexmkrc Detection}

The mode is detected in \texttt{.latexmkrc} by reading the
\texttt{BUILD\_MODE} environment variable:

\begin{minted}{perl}
my $build_mode = $ENV{'BUILD_MODE'} || 'dev';

if ($build_mode eq 'prod') {
    $max_repeat = 7;
} elsif ($build_mode eq 'ultra') {
    $max_repeat = 1;
    $bibtex_use = 0;
    $do_gls = 0;
} else {
    $max_repeat = 6;
}
\end{minted}

The \texttt{\$max\_repeat} variable controls how many times
\texttt{latexmk} reruns \texttt{lualatex} before declaring the build
complete.  Complex documents with many cross-references may need all
seven passes in production mode.

\section{Custom Configuration}

To add a custom build mode, extend \texttt{.latexmkrc}:

\begin{minted}{perl}
if ($build_mode eq 'thesis') {
    $max_repeat = 8;
    $biber = "biber --validate-datamodel --debug %O %S";
    $do_gls = 1;
}
\end{minted}

Define a corresponding macro in the document preamble:

\begin{minted}{latex}
\ifdefined\OmniThesis
  \usepackage{todonotes}
  \setlength{\marginparwidth}{2cm}
\fi
\end{minted}
